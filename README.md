# Quasi-Monte Carlo for a real Bayesian estimation

A reproducible, pure–Wolfram-Language study of **quasi-Monte Carlo
(QMC)** integration — Sobol' and Halton low-discrepancy sequences —
against **plain Monte Carlo (MC)** on an honest Bayesian
parameter-inference problem: a **6-parameter two-compartment
pharmacokinetic (PK) model** fit to the classic **Theophylline** data
set.

The point of the project is to show, empirically and on a problem that
actually matters, the textbook convergence gap

$$\text{QMC error} = O\!\big(N^{-1}(\log N)^{d}\big)
   \qquad\text{vs}\qquad
   \text{MC error} = O\!\big(N^{-1/2}\big),$$

and to turn it into a **practical pay-off**: when every integrand
evaluation is an expensive forward model solve, QMC reaches a target
posterior accuracy with roughly an **order of magnitude fewer model
evaluations**.

## What the project does

1. **Pulls a real data set** — the Theophylline pharmacokinetic study
   (12 subjects, single oral dose, 11 serum-concentration samples each
   over ~25 h), the canonical teaching data set for PK modelling,
   from a public mirror of R's `datasets::Theoph`.
2. **Builds a genuine Bayesian model** — a two-compartment oral PK
   model with first-order absorption (closed-form, no ODE solver
   needed), six parameters {ka, CL, V₁, Q, V₂, σ}, lognormal priors,
   and a Gaussian residual.
3. **Sets up the integration problem correctly.** A diffuse 6-D prior
   used directly as an importance proposal collapses (effective sample
   size ≈ 1). We instead build an **adaptive Gaussian proposal**
   (Laplace mode + a short population-Monte-Carlo refinement of the
   posterior mean and covariance in log-parameter space). This is what
   real QMC-for-Bayes implementations do, and it makes the QMC
   integrand smooth.
4. **Implements the sequences from scratch** — Halton (radical
   inverses) and Sobol' (Joe-Kuo direction numbers, Gray-code
   construction), with randomized variants (Sobol' digital shift,
   Halton Cranley-Patterson rotation) so that root-mean-square error is
   estimable.
5. **Demonstrates the convergence law** — RMSE vs N for the model
   evidence and the posterior parameter means, on log-log axes, with
   fitted slopes ≈ −1 (QMC) vs ≈ −½ (MC).
6. **Shows the dimension dependence** — the $(\log N)^d$ factor, by
   repeating the study at effective dimension d = 2, 4, 6.
7. **Confirms correctness on synthetic truth** — the QMC posterior
   recovers known parameters used to generate synthetic data.
8. Produces a self-contained **Wolfram Community notebook** as the
   end-product (see `community/`).

## Repository layout

The analysis pipeline (data fetch → model → proposal → convergence
study → notebook build) is **pure Wolfram Language**. All figures are
generated by Wolfram Language; there are no external or AI-generated
images.

| path                              | what lives there                                                       |
| ---                               | ---                                                                    |
| `wolfram/fetch_theoph.wls`        | live fetch of the Theophylline data into `data/theoph.csv`             |
| `community/qmc_bayes_helpers.wl`  | the single canonical package (generators, PK model, Bayes, estimators) |
| `wolfram/hero.wls`                | the cover animation (MC vs Sobol' convergence race → `hero.gif`)       |
| `wolfram/classical.wls`           | classical NLS vs Bayesian/QMC comparison figure                        |
| `wolfram/*.wls`                   | figure renderers (sequences, convergence, dimension, fit, payoff)      |
| `wolfram/run_all.wls`             | one entry point, writes `docs/images/*.png`                            |
| `data/`                           | the tidy Theophylline CSV (committed for reproducibility)              |
| `community/`                      | the buildable Wolfram Community notebook + its `.wls` source           |
| `docs/images/`                    | rendered figures referenced from the notebook and the README          |
| `tests/`                          | sanity checks (Sobol' equidistribution, PK conservation, slopes)       |

## Reproducing

```sh
# 1. Fetch the real data (writes data/theoph.csv)
wolframscript -file wolfram/fetch_theoph.wls

# 2. Run the analysis + render every figure (writes docs/images/)
wolframscript -file wolfram/run_all.wls

# 3. Build the community notebook (writes community/qmc_bayes.nb)
wolframscript -file community/build_notebook.wls
```

Wolfram Language 14+ (developed on 15.0). No internet is needed after
step 1, because `data/theoph.csv` is committed.

## Why a pharmacokinetic model?

PK/PD parameter inference is a textbook setting where each likelihood
evaluation requires solving the forward model, the parameter dimension
is modest (≈ 4–10), and posterior expectations (clearance, half-life,
exposure AUC, dosing predictions) are the actual deliverable. It is
exactly the regime — smooth integrand, moderate dimension, expensive
evaluations — where QMC's better-than-$\sqrt N$ convergence pays off in
fewer model runs for the same accuracy.
